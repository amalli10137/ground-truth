# GROUND TRUTH

A signal-recovery / statistical-inference game. Every level shows a dataset generated by a
hidden equation or stochastic process; you recover the generator with real math, in **Python,
in the browser** (Pyodide + numpy/scipy/pandas in a web worker). Overlay your candidate curves
on the observed data, watch the residuals, and submit when the noise looks like noise.

23 levels across four tiers:

| Tier | Levels | Teaches |
|---|---|---|
| FOUNDATIONS | 1–7 | sample mean, OLS, FFT, coherent integration, random walks, AR(1), GBM |
| WICKED | 8–12 | spectral leakage, chirps/Hilbert, matrix pencil, Wiener deconvolution, compressed sensing |
| STOCHASTIC HORRORS | 13–19 | chaos vs noise, α-stable tails, jump-diffusion, Hurst exponents, HMM/Baum–Welch, Kalman, Hawkes |
| QUANT GAUNTLET | 20–23 | cointegration, random matrix theory, multiple-testing, distributional forecasting (final boss) |

## Run it

```bash
npm install
npm run dev        # level packs are committed in public/levels/
```

The first "Run" in the code lab boots Pyodide from CDN (~40 MB of numpy/scipy/pandas,
first time only — cached by the browser after that).

## Regenerating level data

Level packs (`public/levels/NN.json`) are produced by the Python `datagen/` package with fixed
seeds, so data and answers are deterministic and the grader is exact. Grading tolerances are
**~4 standard errors of the intended estimator**, computed analytically or by Monte Carlo —
never hand-tuned.

```bash
python3 -m venv .venv && .venv/bin/pip install numpy scipy pandas pytest
.venv/bin/python -m datagen.build          # rebuild all packs
.venv/bin/python -m pytest datagen/tests   # winnability suite (see below)
```

## Testing

- **Winnability** (`datagen/tests/`): for every level, the *intended* estimator runs on the
  exact shipped JSON and must land inside the shipped tolerance. If a seed produces an outlier
  draw, the rule is: change the seed, not the tolerance. The suite also verifies the traps
  (naive peak reading is biased on level 8, unregularized deconvolution explodes on 11,
  realized variance is jump-contaminated on 15, a flat-vol forecast fails the final boss, …).
- **Frontend unit tests**: `npm test` — grading functions (parameter/function/pinball),
  worker protocol with a stubbed worker.
- **Bridge integration**: `npm run test:integration` — boots real Pyodide under Node and
  exercises `data`/`overlay`/`overlay_fn`/`residuals`/`plot_xy`, error surfacing, the
  `predict(t)` grading path, and namespace reset.
- **Browser E2E**: `npm run test:e2e` (with `npm run dev` running and Playwright's Chromium
  installed) — loads the app, boots Pyodide in the worker, runs the level-1 starter code,
  checks overlays/residuals/console, forces a Python exception, submits the answer, and
  verifies the DECLASSIFIED stamp, unlock, and persistence across reload.

## Architecture

- React 18 + Vite + TypeScript; uPlot charts (graph-paper theme, overlays and regime shading
  drawn via canvas hooks); CodeMirror 6 editor; KaTeX for the derivation hints.
- Pyodide runs in a dedicated web worker (`src/worker/`); the Python-side bridge
  (`src/worker/bridge.py`) exposes `data`, `overlay`, `overlay_fn`, `residuals`, `shade`,
  `plot_xy`, `clear_overlays` to player code. A Python exception surfaces in the console
  panel; it never crashes the page. Stop hard-terminates and reboots the runtime.
- Three grading modes: **parameter** (target ± tol or range fields), **function**
  (`predict(t)` scored by RMSE on a hidden extension of the time axis — level 10), and
  **distribution** (quantiles scored by pinball loss against 10,000 hidden Monte-Carlo
  draws — level 23).
- Progress, per-level code drafts, hint usage and attempts persist in `localStorage`.
  Case record screen shows par (attempts + hints) and exports your solved-level code as a
  single annotated `.py` cookbook.
